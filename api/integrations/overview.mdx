## Choosing an Integration Type

<AccordionGroup>
  <Accordion title="Incoming Webhooks - Get data into Zulip">
    Best for:
    - Receiving notifications from external services (GitHub, GitLab, etc.)
    - Automated alerts and monitoring
    - CI/CD pipeline notifications
    - One-way data flow into Zulip
    
    Examples:
    - GitHub pull request notifications
    - Datadog alerts
    - Deployment notifications
  </Accordion>
  
  <Accordion title="Outgoing Webhooks - Send data from Zulip">
    Best for:
    - Triggering actions when messages are sent
    - Command-based integrations (@bot do something)
    - Forwarding messages to external systems
    - Two-way communication
    
    Examples:
    - Translation bots
    - Task creation in external systems
    - Custom slash commands
  </Accordion>
  
  <Accordion title="Bots - Interactive applications">
    Best for:
    - Complex interactive features
    - Stateful conversations
    - Real-time responses to messages
    - Full access to Zulip data
    
    Examples:
    - Help desk bots
    - Poll/survey bots
    - Game bots
    - Custom commands and workflows
  </Accordion>
  
  <Accordion title="REST API - Full control">
    Best for:
    - Custom applications
    - Data synchronization
    - Administration and user management
    - Building custom clients
    
    Examples:
    - Analytics dashboards
    - User provisioning
    - Custom mobile/web apps
    - Data exports
  </Accordion>
</AccordionGroup>

## Quick Start Guides

<Steps>
  <Step title="Choose Integration Type">
    Decide which integration pattern fits your use case best.
  </Step>
  
  <Step title="Create Bot Account (if needed)">
    For outgoing webhooks and bots, create a bot user in Zulip settings.
  </Step>
  
  <Step title="Get Credentials">
    Obtain API key or webhook URL from Zulip.
  </Step>
  
  <Step title="Configure External Service">
    Set up the external service to send data to Zulip or receive from it.
  </Step>
  
  <Step title="Test Integration">
    Send test data to verify the integration works.
  </Step>
</Steps>

## Best Practices

<Check>
  **Use bot accounts** for integrations instead of personal accounts
</Check>

<Check>
  **Channel organization** - Create dedicated channels for integration notifications
</Check>

<Check>
  **Message formatting** - Use Zulip's Markdown for rich, readable messages
</Check>

<Check>
  **Error handling** - Implement retries and graceful failures
</Check>

<Check>
  **Rate limiting** - Respect API rate limits and implement backoff
</Check>

<Warning>
  Store API keys securely. Never commit them to version control.
</Warning>
