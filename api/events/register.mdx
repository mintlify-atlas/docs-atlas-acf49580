## Endpoint

```http
POST /api/v1/register
```

Register an event queue to receive real-time updates from Zulip.

<Info>
  The events API is how Zulip's web and mobile apps get real-time updates. It powers all live features in the product.
</Info>

## Overview

The Zulip events system works in two steps:

1. **Register**: Create an event queue with this endpoint
2. **Poll**: Fetch events from the queue using [GET /api/v1/events](/api/events/get)

The queue receives events for all changes relevant to the authenticated user.

## Request Parameters

<ParamField body="event_types" type="array">
  Array of event types to subscribe to.
  
  Common types:
  - `"message"` - New messages
  - `"subscription"` - Channel subscription changes
  - `"realm_user"` - User changes
  - `"presence"` - User presence updates
  - `"update_message"` - Message edits
  
  If not specified, subscribes to all event types.
</ParamField>

<ParamField body="narrow" type="array">
  Narrow filter for message events (same format as [GET /messages](/api/messages/get)).
  
  Example: `[["stream", "general"]]` for messages in the general channel.
</ParamField>

<ParamField body="all_public_streams" type="boolean" default="false">
  Whether to include all public channels in initial data.
</ParamField>

<ParamField body="include_subscribers" type="boolean" default="false">
  Whether to include subscriber lists for channels.
</ParamField>

<ParamField body="client_gravatar" type="boolean" default="false">
  Whether to use client-side Gravatar rendering.
</ParamField>

<ParamField body="apply_markdown" type="boolean" default="true">
  Whether message content should be in HTML (true) or Markdown (false).
</ParamField>

<ParamField body="fetch_event_types" type="array">
  Event types to include in the initial state fetch.
</ParamField>

## Response

<ResponseField name="queue_id" type="string" required>
  The ID of the event queue. Use this to fetch events.
</ResponseField>

<ResponseField name="last_event_id" type="integer" required>
  The ID of the last event. Use -1 initially when fetching events.
</ResponseField>

<ResponseField name="zulip_feature_level" type="integer">
  The server's feature level for API compatibility.
</ResponseField>

<ResponseField name="zulip_version" type="string">
  The Zulip server version.
</ResponseField>

<ResponseField name="realm_uri" type="string">
  The organization's base URL.
</ResponseField>

```python Python - All Events
import zulip
import sys

client = zulip.Client(config_file="~/.zuliprc")

# Handle all events
def handle_event(event):
    if event['type'] == 'message':
        msg = event['message']
        print(f"New message: {msg['sender_full_name']}: {msg['content']}")
    elif event['type'] == 'subscription':
        print(f"Subscription change: {event['op']}")
    elif event['type'] == 'presence':
        print(f"Presence update: User {event.get('user_id')} is {event.get('presence', {}).get('aggregated', {}).get('status')}")
    else:
        print(f"Event: {event['type']}")

print("Listening for all events...")
client.call_on_each_event(handle_event)
```

## Event Types

Common event types you can subscribe to:

<AccordionGroup>
  <Accordion title="message">
    New messages sent in channels or direct messages.
    
    Essential for chat bots and real-time message processing.
  </Accordion>
  
  <Accordion title="subscription">

## Queue Lifecycle

1. **Register**: Create queue with `POST /register`
2. **Poll**: Repeatedly call `GET /events` with the queue_id
3. **Timeout**: Queues automatically expire after ~10 minutes of inactivity
4. **Reconnect**: If queue expires, register a new one

```python Queue Management
def maintain_event_queue():
    client = zulip.Client(config_file="~/.zuliprc")
    
    while True:
        try:
            # Register new queue
            result = client.register(event_types=["message"])
            queue_id = result['queue_id']
            last_event_id = result['last_event_id']
            
            # Poll for events
            while True:
                events_result = client.get_events(
                    queue_id=queue_id,
                    last_event_id=last_event_id
                )
                
                for event in events_result['events']:
                    process_event(event)
                    last_event_id = max(last_event_id, event['id'])
                    
        except Exception as e:
            print(f"Queue error: {e}, reconnecting...")
            time.sleep(1)
```

## Best Practices

<Check>
  **Use client libraries** - They handle queue management, reconnection, and exponential backoff automatically.
</Check>

<Check>
  **Filter event types** - Only subscribe to events you need to reduce bandwidth.
</Check>

<Check>
  **Handle reconnection** - Queues expire; implement automatic reconnection logic.
</Check>

<Check>
  **Process events quickly** - Don't block the event loop with slow processing.
</Check>

<Warning>
  Long-polling requests can timeout. Use the `event_queue_longpoll_timeout_seconds` from the response as your HTTP timeout.
</Warning>
